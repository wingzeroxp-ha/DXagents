# 🤖 有极智能 Agent — 使用说明

> **一句话：一个 U 盘就能带走的双引擎 AI Agent 工作台**
> 离线可用 · 零安装 · 免配置 Node/Python 环境

---

## 📦 快速开始

### Windows

| 步骤 | 操作 |
|------|------|
| ① | 插入 U 盘 / 进入 `AI-Agent-Portable` 目录 |
| ② | 打开终端（cmd 或 PowerShell），运行： |
|    | `node launcher/server.js` |
| ③ | 浏览器自动打开 → [http://127.0.0.1:3800](http://127.0.0.1:3800) |
| ④ | 看到启动器首页，选择模式开始使用 |

> ⚠️ 如果 3800 被占用，启动器会自动切换到 3801/3802… 请以控制台输出的实际端口为准。

### Mac (Apple Silicon)

| 步骤 | 操作 |
|------|------|
| ① | 插入 U 盘 / 进入 `AI-Agent-Portable` 目录 |
| ② | 打开终端，运行： |
|    | `node launcher/server.js` |
| ③ | 浏览器打开 → 控制台输出的地址 |

> 💡 首次启动会自动解压内置 Node.js，无需提前安装。

## 💬 对话功能详解

### 输入方式

| 操作 | 效果 |
|------|------|
| `Enter` | 发送消息 |
| `Shift + Enter` | 换行 |
| 📎 按钮 | 添加图片或文本文件（最大 2MB，限 4 个） |
| 📋 按钮 | 复制助手回复 |

### 流式输出

助手回复会**逐字显示**（打字机效果），无需等待完整响应。你可以：

- 在回复过程中随时**发送新消息** — 自动中断上一轮
- 点击 **✕** 清空对话 — 自动存档后再清空

### 智能体预设

内置 19 个预置角色，也支持**用户自定义智能体**：

在「智能体」面板：
- **预置智能体** — 选择一个内置角色，点击 **使用** 注入 systemPrompt
- **自定义智能体** — 点击 **创建自定义智能体**，填写：

| 字段 | 说明 |
|------|------|
| 名称 | 自定义智能体名称 |
| 图标 | 从 emoji 列表选择 |
| 描述 | 简短说明（可选） |
| 系统提示词 | 设定角色、行为规则、输出格式 |
| 能力标签 | 逗号分隔，如 `code_review, debug` |
| 关联模型 | 激活时自动切换到该模型 |

自定义智能体可以随时编辑或删除，激活时自动切换关联模型。

```
┌──────────────────────────────────────┐
│  🤝 销售顾问                        │
│  专业的销售支持、客户沟通与谈判策略   │
│  [customer_analysis] [communication] │
│         [👉 使用]                     │
└──────────────────────────────────────┘
```

点击 **使用** 后：
- 该角色的 systemPrompt 注入到后续对话
- 对话顶部显示当前预设名称（带清除按钮）
- 预设可随时切换或清除

### 后端自动切换

```
Agent（OpenClaw/Hermes）正在运行
        ↓
  聊天 → Agent 自身 API → 完整工具能力
        ↓
Agent 未运行
        ↓
  聊天 → 直连模型 API → 快速对话
```

## ⚙️ 配置模型

### 第一步：获取 API Key

从模型厂商（DeepSeek、OpenAI、Claude、通义千问 等）获取 API Key。

### 第二步：配置

1. 进入工作模式或专业模式 →「API 配置」面板
2. 点击模型市场中的任意模型卡片
3. 在弹出的对话框中，填写该**服务商**的配置：
   - API Key（同一服务商只需填一次）
   - Base URL
   - 勾选该服务商下可用的模型（如 DeepSeek-V3、DeepSeek-R1 等）
4. 保存后在对话界面可通过下拉菜单切换该服务商的任意模型

> 配置以**服务商**为单位：一个 API Key 对应一个服务商下的所有模型，
> 避免了为每个模型重复填写相同 API 地址和 Key 的麻烦。

### 模型市场

提供 **28 个模型** 参考信息（国产为主 + 国外主流），包含：

```
┌─────────────────────────────────┐
│ DeepSeek-V3       DeepSeek     │
│ 📝文本 🗣️语音 🖼️识图            │
│ ✅ 已配置 / ⚪ 未配置            │
│ 性价比：★★★★★                  │
│ [注册链接]  [充值链接]           │
└─────────────────────────────────┘
```

点击 **删除 Key** 可清除当前配置对象的 API Key。

## 🔧 维护与复位

### 备份 / 还原

在「维护」面板操作：
- **备份**：打包配置（config/）、用户数据（data/openclaw/、data/hermes/）、Agent 脚本目录（agents/ 跳过 node_modules 和运行时二进制）、启动器代码（launcher/）
- **还原**：恢复最新备份（备份不包含 node_modules，还原后需重新运行对应平台 Agent）
- **清缓存**：清理 Agent 缓存（不删除配置和记忆）

> ⚡ v2.0 起备份跳过 `node_modules` 和 `hermes-agent` 等运行时文件，大幅提升速度（~7s），备份体积控制在 ~45MB。Agent 运行时可在还原后通过重新启动自动恢复。

### 一键复位

> 在 `AI-Agent-Portable/` 根目录

| 系统 | 文件 | 清除内容 | 保留内容 |
|------|------|----------|----------|
| Windows | `一键复位.bat` | 模型配置、权限配置、语言设置、运行时记录 | Agent 核心、预设角色、模型市场、启动器 |
| Mac | `一键复位.command` | 同上 | 同上 |

## 📁 目录结构

### 开发目录（含 Git 仓库，不放入 U 盘）

```
AI-Agent-Portable/                ← Git 仓库根目录
├── .git/                         ← Git 版本记录（部署时排除）
├── .gitignore                    ← 忽略 runtime/、data/、backup-*/ 等
├── launcher/                     ← 启动器源码（核心，需提交 Git）
│   ├── server.js                 ← HTTP 服务入口
│   ├── package.json
│   ├── lib/                      ← 后端模块（config/agent/backup/sse…）
│   └── public/                   ← 前端（index.html / app.js / workspace.js / style.css / modules/）
├── config/                       ← 预置配置（presets.json / models.json / providers.json 等，已提交 Git）
├── agents/                       ← Agent 运行核心（openclaw / hermes，运行时依赖不提交）
├── data/                         ← 运行时数据（日志/备份/记忆，gitignore 排除）
├── runtime/                      ← 内置 Node/Python 运行环境（gitignore 排除）
├── docs/                         ← 本文档
├── 一键复位.bat / .command       ← 恢复出厂设置
└── Windows-启动.bat / Mac-启动.command ← 启动启动器（node launcher/server.js）
```

### U 盘部署目录（即整个 AI-Agent-Portable，剔除以下内容）

部署到第二台电脑 / U 盘时，复制 **整个 `AI-Agent-Portable` 目录**，但无需包含：

| 排除项 | 原因 |
|--------|------|
| `.git/` | 版本记录，约数十 MB，非运行必需 |
| `backup-*/` | 旧的手动备份，已被 git 与运行时备份取代 |
| `data/launcher/backups/` 中历史备份 | 运行时自动管理 |

> 其余目录（`launcher/`、`config/`、`agents/`、`runtime/`、`docs/`、复位/启动脚本）**必须完整复制**，
> 否则 U 盘版本无法离线运行。

### 关键文件说明

| 路径 | 作用 |
|------|------|
| `launcher/public/workspace.js` | 工作台前端逻辑（含智能体面板 `renderPresetsPanel`） |
| `launcher/public/style.css` | 全部样式（`.preset-grid` 等布局规则） |
| `launcher/public/index.html` | 首页 + 样式版本号 `__CSS_VERSION__`（防缓存） |
| `launcher/server.js` | 服务入口，含 `no-store` 缓存头与 CSS 版本注入 |
| `config/presets.json` | 19 个预置智能体定义 |

## 🆘 常见问题

**Q: 浏览器没自动打开？**
A: 手动访问控制台输出的地址（如 `http://127.0.0.1:3800`）

**Q: Agent 启动失败？**
A: 先检查「API 配置」面板是否已填写有效的 API Key

**Q: 模型返回空内容？**
A: 检查 API Key 余额/额度，或确认 Base URL 正确

**Q: 如何更新？**
A: 替换 `launcher/` 和 `config/` 目录，`data/` 和 `agents/` 保留。
未来版本将支持**在线远程更新**：界面、功能代码、模型信息可通过更新服务器自动升级，运行环境（Node/Python 二进制）非必要不更新。

## 🐞 已知问题与修复记录

### v2.0.1 — 智能体面板网格错位（对角线瀑布流）

**现象**：「智能体」面板的卡片呈对角线级联、互相重叠，而非正常网格排列。

**根因**：`launcher/public/workspace.js` 的 `presetCard()` 函数生成卡片 HTML 时，
**漏写了一个 `</div>`** 闭合标签（`.preset-card` 未闭合）。导致每张卡片被嵌套进上一张卡片内部，
CSS Grid 布局彻底失效。

**修复**（已合入）：

1. `workspace.js` 补回缺失的 `</div>`，使 `.preset-card` 正确闭合；
2. `style.css` 恢复 `.preset-grid { padding: 16px 20px }`（与正常版本一致）；
3. `style.css` 将 `.panel-wrap` 由 `min-height: 100%` 改回 `height: 100%`；
4. `style.css` 移除 `.preset-desc` 上多余的 `-webkit-line-clamp` 限制；
5. `server.js` 增加 `Cache-Control: no-store` + `index.html` 中 `__CSS_VERSION__` 时间戳，规避浏览器缓存导致的「改了不生效」。

> 排查过程中曾误判为浏览器缓存（已用无痕模式 + 强制刷新 + 内联 `!important` 排除），
> 最终通过对比 `backup-pre-pro` 正常版本定位到是**结构标签缺失**而非样式问题。

## 🔄 更新与同步

### 开发机 → Git → U 盘 的标准流程

1. 在开发目录改代码 → 本地自测（刷新浏览器验证智能体面板为正常网格）；
2. `git add -A && git commit && git push` 做版本备份；
3. 将 `AI-Agent-Portable`（排除 `.git/`、`backup-*/`）整体复制到 U 盘；
4. 第二台电脑从 U 盘运行 `node launcher/server.js`。

> ⚠️ 因第二台电脑无 Git，U 盘复制是唯一同步手段；请确保复制的是**最新提交后**的目录。
