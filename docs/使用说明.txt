AI Agent 便携启动包使用说明 v2.0

一、根目录入口

Windows 用户：
  双击 一键复位.bat       — 还原配置到出厂状态（不清除 Agent 核心/预设/模型市场）
  启动器默认自动运行       — 执行 launcher/server.js，监听 http://127.0.0.1:3800
  （暂无可双击的启动脚本，需手动运行 node launcher/server.js）

Mac M 芯片用户：
  双击 一键复位.command    — 还原配置到出厂状态
  启动器默认自动运行       — 同上

启动器端口：3800-3900（自动避让已占用端口）

二、界面结构

  ?/                    — Launcher Shell 模式选择页
  simple.html           — 简易模式（原版直连模型 API 对话）
  #work                 — 工作模式（进阶 SPA 界面）
  #pro                  — 专业模式（含文件浏览器/日志/API 探索）
  #work/openclaw        — 使用 OpenClaw Agent 的工作模式
  #work/hermes          — 使用 Hermes Agent 的工作模式

三、  工作模式（#work）

   左侧边栏：导航切换以下面板
     ─ 对话：Markdown 气泡式聊天，支持图片/文件附件、复制、清空、
       SSE 打字机流式输出（逐字显示回复），发送新消息时自动中断上一轮
     ─ 预置智能体：10 个预设角色（销售顾问/管理助理/风格顾问等），
       点击"使用"注入对应 systemPrompt 到对话，对话顶部显示预设名称
       支持用户创建自定义智能体（名称/图标/提示词/技能标签/关联模型）
     ─ 历史：浏览/加载/删除历史会话，清空对话时自动存档
      ─ 模型市场：国产 20 模型 + 国外 8 模型，点击任意模型卡片配置
       该服务商（填写一次 API Key + Base URL 即可使用该服务商下所有模型）
     ─ 权限范围：安全/工作区/高级三种权限模式
     ─ 诊断中心：运行环境检测
     ─ 维护：备份/恢复/清缓存

  专业模式（#pro）额外面板：
     ─ 文件浏览器：浏览 Agent 工作目录文件
     ─ 实时日志：SSE 推送 Agent 运行日志
     ─ API 探索：调用启动器各 API 端点
     ─ Raw 模式：对话中显示完整 JSON 请求/响应

  后端自动逻辑：
     Agent（OpenClaw/Hermes）正在运行 → 聊天走 Agent 自身 API
     Agent 未运行 → 自动回退到直连模型 API（simple-chat 模式）

四、配置存储

  模型配置：    config/model.json / config/model.openclaw.json / config/model.hermes.json
                （每个服务商一条记录，包含该服务商下所有模型 ID）
  预置智能体：  config/presets.json（10 个内置预设角色）
  自定义智能体：config/custom-presets.json（用户创建的智能体）
   服务商定义：  config/providers.json（22 个模型服务商的 API 元信息）
   模型市场：    config/models.json（国产 20 模型 + 国外 8 模型）
  运行端口：    config/runtime-ports.json
  对话历史：    data/hermes/ / data/openclaw/ 下的 simple-chat-*.json

  复位工具保留：
     agents/（Agent 核心代码）
     config/presets.json（内置预设）
     config/providers.json（服务商定义）
     config/models.json（模型市场）
     launcher/（启动器）

  复位工具清除：
     config/model-*.json / config/model.json（模型配置）
     config/custom-presets.json（自定义智能体）
     config/permissions.json
     config/language.json
     data/hermes/ + data/openclaw/ 下的运行时记录
     launcher 运行时记录

五、重要原则

  1. 不依赖电脑里已安装的 Node.js（使用内置 runtime/node-win-x64 或 node-mac-arm64）
  2. 不启动电脑里已有的 OpenClaw 或 Hermes
  3. 只启动 AI-Agent-Portable/agents 目录里的便携 Agent
  4. 只停止本启动器自己启动的 Agent
  5. OpenClaw 和 Hermes 的数据、记忆、日志、workspace 互相隔离

六、数据目录

  OpenClaw：        data/openclaw
  Hermes：          data/hermes
  启动器运行时：    data/launcher

七、Windows 提醒

  Windows 可能出现防火墙或杀毒软件提醒，这是因为本程序会启动 127.0.0.1 本地服务。
  如果要正常使用，请允许本程序在本机运行。
  Hermes 内置的浏览器自动化引擎体积较大，部分杀毒软件可能会额外扫描，首次启动慢
  一点属于正常现象。

八、Mac 提醒

  Mac 可能提示无法验证开发者，或者要求文件访问、辅助功能、屏幕录制等权限。
  默认安全模式只访问 U 盘目录，只有启用高级工具时才需要更多权限。
  如果双击 .command 提示没有权限，请在终端进入 U 盘根目录后执行：
  chmod +x Mac-启动.command
  如果 Mac 没有 Mac-启动.command，请手动运行 node launcher/server.js

  Mac 首次启动会做初始化，时间可能较长：
  OpenClaw 首次安装依赖通常需要几分钟。
  Hermes 首次生成虚拟环境和安装依赖可能需要更久，并且需要网络访问 Python 依赖源。
